🔘 CTA Design & Placement

8 ▼
Current State

The "Checkout" button uses a full-width primary button style with a lock icon SVG (signaling security), proper disabled state when cart is empty, and the button is disabled during cart updates to prevent stale-total submissions. On mobile, the cart page footer is sticky so the CTA is always visible.

  • Lock icon builds security trust
  • Full-width button (high tap target)
  • Sticky footer on mobile (always visible)
  • Disabled during updates (prevents errors)
  • Cart drawer has single clear CTA
  • No visual urgency (color, animation)
  • No secondary CTA (e.g., Apple Pay / Google Pay express)
  • Cart page CTA not above the fold on desktop
How to Improve
1 Add dynamic checkout buttons (Apple Pay, Google Pay, Shop Pay) — the code references additional_checkout_buttons but they may not be enabled in Shopify settings. Enable them to give customers express checkout with 1 tap.
2 Consider a contrasting CTA color that stands out from the rest of the theme. The button should be the most visually dominant element in the footer area.
3 On desktop, add a mini sticky checkout bar that appears as the user scrolls past the main CTA.

🏷️ Discount & Promo Experience

7 ▼
Current State

Both cart drawer and cart page have a collapsible "Have a discount code?" field with a tag icon. When a code is entered, it redirects to /checkout?discount=CODE to pre-apply the discount. Line-level and cart-level discount applications are displayed with the discount icon and title. The savings summary aggregates all savings into one banner.

  • Discount field on both drawer and page
  • Collapsible (doesn't clutter by default)
  • Applied discounts shown with names
  • Savings summary reinforces value
  • No inline validation (user must go to checkout to see if code works)
  • No error message if code is invalid
  • No "remove discount" option
  • Redirect to checkout breaks flow if code fails
How to Improve
1 Use Shopify's AJAX discount API to validate the code in-place instead of redirecting to checkout. Show a green success message or red error inline — this prevents cart abandonment from failed codes.
2 Add a loading spinner on the "Apply" button while validating to show the user something is happening.
3 If you run frequent promotions, consider showing an auto-applied discount banner (e.g., "FAMILY20 auto-applied — 20% off!") when automatic discounts are active.

🛍️ Cross-sell & Upsell

7 ▼
Current State

Cart drawer shows "You may also like" with 3 products from the first cart item's collection. Cart page shows "Complete the Look" with 4 products in a grid. Both exclude items already in cart. Products link to their PDPs. Images are lazy-loaded with hover zoom on cart page.

  • Cross-sell on both drawer and page
  • Smart filtering (excludes items in cart)
  • Collection-based relevance
  • Clean, compact presentation
  • Hover effects for engagement
  • No "Add to Cart" button (must visit PDP)
  • Only uses first item's collection (ignores rest)
  • No Shopify product recommendations API
  • No "frequently bought together" bundles
How to Improve
1 Add a quick "Add to Cart" button on each cross-sell item — requiring a PDP visit adds friction and most users won't click through. For your mommy-and-me products, this is especially important since customers often need matching items.
2 Use Shopify's Product Recommendations API (/recommendations/products.json?product_id=X) for more intelligent suggestions based on purchase patterns rather than just collection membership.
3 Consider a "Complete the Set" bundle offer with a small discount (e.g., "Add matching daughter dress — save 10%").

⚠️ Error Handling

8 ▼
Current State

Error handling differentiates between Shopify API errors and network failures. Stock limit messages are specific ("Only 3 available — we've updated your quantity") rather than generic. Network errors show a clear connection message. Errors display inline next to the affected line item with a red error icon. The checkout button is disabled during updates to prevent stale-total submissions.

  • Specific stock-limit messages
  • Network vs. API error differentiation
  • Inline errors per line item
  • ARIA role="alert" on error containers
  • Checkout disabled during updates
  • No auto-retry on network failure
  • Error text is small and easy to miss
  • No "out of stock" visual state for items
How to Improve
1 Add an auto-retry mechanism for transient network errors (retry once after 2 seconds before showing the error message).
2 If an item goes out of stock while in the cart, show a prominent visual state (greyed out image, "Sold Out" overlay) rather than just a small error message.
3 Add a "Notify me when back in stock" option for out-of-stock items in the cart.

How to Improve

⚡ Loading & Performance

8 ▼
Current State

Cart updates use AJAX fetch with section rendering (Shopify's most efficient method). Scripts are defer-loaded. Loading spinners appear on affected line items during quantity changes. Images use lazy loading. The cart form uses the critical-hidden class pattern to prevent layout shifts. Cart note uses debounced input.

  • AJAX section rendering (fast updates)
  • Deferred script loading
  • Per-item loading spinners
  • Lazy loaded images
  • Debounced inputs
  • PubSub event system for coordinated updates
  • No optimistic UI updates (waits for server)
  • Cross-sell images load all at once
  • No skeleton loading states
How to Improve
1 Implement optimistic UI updates — update the quantity and price immediately in the UI while the AJAX request processes, then reconcile if the server response differs. This makes the cart feel instant.
2 Add skeleton loading placeholders for the cross-sell section to prevent layout shift on slower connections.

Cart tables use proper semantic markup (thead, tbody, th with scope, role="table/row/rowgroup"). Visually hidden labels exist for quantity inputs.   • Semantic table markup
  • ARIA live regions for cart updates
  • Focus trapping in cart drawer
  • Descriptive aria-labels on buttons
  • role="alert" on error containers
  • aria-expanded on collapsible sections
  • Discount code field uses inline onclick (not progressive enhancement)
  • Cross-sell items lack descriptive aria-labels

How to Improve
1 Add aria-hidden="true" to all decorative SVG icons (trust badges, discount tag icon, savings tag icon) to reduce screen reader noise.

4 Refactor the discount code toggle from inline onclick to a proper JavaScript event listener with progressive enhancement.

⏰ Urgency & Scarcity

3 ▼
Current State

There are currently no urgency or scarcity elements in the cart experience. No low-stock indicators, no time-limited offers, no cart reservation timers, and no "X people are viewing this" indicators. The only urgency signal is the stock-limit error that appears after a user tries to add more than available — but this is reactive, not proactive.

  • No fake urgency (honest approach)
  • No low-stock warnings on items
  • No delivery date estimate
  • No limited-time offer display
  • No cart expiry / reservation timer
  • No "selling fast" indicators
How to Improve
1 Add real low-stock indicators using item.variant.inventory_quantity. When stock is below a threshold (e.g., 5), show "Only 3 left!" next to the item. This is truthful and creates genuine urgency.
2 Add an estimated delivery date (e.g., "Order within 2 hrs, get it by March 3"). This is one of the highest-impact conversion elements — it answers "when will I get this?" which is a top customer concern.
3 If you run time-limited sales, show a countdown timer in the cart when a sale discount is applied (e.g., "Sale ends in 4h 23m").
